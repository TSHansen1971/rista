# RISTA-001 – Scope and Principles

## Purpose

Rísta shall be a small Apple Silicon-native macOS Markdown editor for open `.md` files.

The core principle is simple: the user writes plain text and sees the document rendered live, without leaving the keyboard and without being locked into a proprietary document format.

## Product boundary

Rísta is not a new writing platform. It is a focused local tool for writing and editing Markdown files on modern macOS.

The product should feel native to macOS, remain fast on Apple Silicon, and avoid unnecessary application layers, accounts, cloud dependencies or opaque storage formats.

## Non-goals

The first phase shall not introduce:

- user accounts
- cloud sync
- database-backed document storage
- AI features
- embedded publishing platform
- proprietary document format
- Markdown Pro code, icons, screenshots, App Store text or visual identity
- Rosetta or Intel compatibility as a design dependency

## Local development rule

The project shall be buildable, runnable, testable and version-controlled locally before any distribution decision is made.

## Repository language rule

General repository material shall be written in English for international readability. Norse terminology is allowed where it belongs to the name, brand or linguistic rationale. Norwegian is allowed where explicitly intended.
